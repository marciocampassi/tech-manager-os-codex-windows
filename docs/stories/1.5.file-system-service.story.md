## Dev Notes

### Previous Story Insights (from Story 1.4)

- **ESM is active**: all source imports need `.js` extensions; test imports are resolved by `moduleNameMapper` in Jest config so `.js` still works in tests.
- **`jest.unstable_mockModule` is required for ESM mocks** — `jest.mock` does NOT work with ESM. Declare mocks before any `await import()` of the module under test.
- **Dynamic import pattern**: after `jest.unstable_mockModule('fs-extra', () => ({ ... }))`, use `const { FileSystemService } = await import('../../src/services/file-system.service.js')`.
- **Typed mock functions**: declare as `jest.fn<() => Promise<void>>()` or `jest.fn<() => Promise<unknown>>()` to avoid TypeScript inferring `never` on `.mockResolvedValue(...)` calls. (Learned in Story 1.4.)
- **`Object.setPrototypeOf` on custom errors**: needed for correct `instanceof` across ESM module boundaries. Pattern established in `AIProviderError` (Story 1.4).
- **Singleton export**: `export const fileSystemService = new FileSystemService()` — consistent with `configService` from Story 1.3. Module-level singleton is acceptable here (no encrypted state, no env-var side effects).
- **`jest.fn<() => Promise<void>>().mockResolvedValue(undefined)`**: use `undefined` (not `void`) for void-returning mock resolutions.

### `fs-extra` API Reference

The following `fs-extra` functions are needed for this story:

```typescript
import * as fs from 'fs-extra';

fs.ensureDir(dirPath: string): Promise<void>          // mkdir -p, no error if exists
fs.writeFile(filePath: string, content: string, encoding: 'utf8'): Promise<void>
fs.readFile(filePath: string, encoding: 'utf8'): Promise<string>
fs.rename(oldPath: string, newPath: string): Promise<void>  // atomic on same filesystem
fs.remove(filePath: string): Promise<void>            // rm -rf, no error if not exists
fs.move(src: string, dest: string, options?: { overwrite?: boolean }): Promise<void>
fs.pathExists(filePath: string): Promise<boolean>     // safe exists check, never throws
```

`fs-extra` v11+ bundles its own TypeScript types — no `@types/fs-extra` needed.

### Atomic Write Pattern

The `writeFile` method must guarantee no partial writes are observable — write to a temp file then rename:

```typescript
async writeFile(filePath: string, content: string): Promise<void> {
  const tmpPath = `${filePath}.${process.pid}.tmp`;
  try {
    await fs.ensureDir(path.dirname(filePath));
    await fs.writeFile(tmpPath, content, 'utf8');
    await fs.rename(tmpPath, filePath);  // atomic on same filesystem
  } catch (err) {
    await fs.remove(tmpPath).catch(() => {});  // best-effort cleanup
    throw new FileSystemError(
      err instanceof Error ? err.message : 'Write failed',
      'writeFile', filePath, err,
    );
  }
}
```

`rename` is atomic on POSIX systems as long as source and destination are on the same filesystem. Using `path.dirname(filePath)` for the temp file guarantees this.

[Source: docs/prd/technical-assumptions.md#performance-constraints — File operations: Atomic writes]

### `FileSystemError` Design

Follow the same pattern as `AIProviderError` (Story 1.4):

```typescript
export class FileSystemError extends Error {
  constructor(
    message: string,
    public readonly operation: string,
    public readonly path: string,
    public readonly cause?: unknown,
  ) {
    super(message);
    this.name = 'FileSystemError';
    Object.setPrototypeOf(this, FileSystemError.prototype);
  }
}
```

### Mocking `fs-extra` in Tests

`fs-extra` is an ESM-compatible module — use `jest.unstable_mockModule`:

```typescript
// tests/services/file-system.service.test.ts
import { describe, it, expect, jest, beforeEach } from '@jest/globals';

const mockEnsureDir = jest.fn<() => Promise<void>>();
const mockWriteFile = jest.fn<() => Promise<void>>();
const mockReadFile = jest.fn<() => Promise<unknown>>();
const mockRename = jest.fn<() => Promise<void>>();
const mockRemove = jest.fn<() => Promise<void>>();
const mockMove = jest.fn<() => Promise<void>>();
const mockPathExists = jest.fn<() => Promise<unknown>>();

jest.unstable_mockModule('fs-extra', () => ({
  ensureDir: mockEnsureDir,
  writeFile: mockWriteFile,
  readFile: mockReadFile,
  rename: mockRename,
  remove: mockRemove,
  move: mockMove,
  pathExists: mockPathExists,
}));

const { FileSystemService, FileSystemError } = await import(
  '../../src/services/file-system.service.js'
);

let service: InstanceType<typeof FileSystemService>;

beforeEach(() => {
  service = new FileSystemService();
  // reset all mocks between tests
  mockEnsureDir.mockReset();
  mockWriteFile.mockReset();
  // ... etc
});
```

### Refactoring Performed

- **File**: `tests/services/file-system.service.test.ts`
  - **Change**: Replaced 4 `try/catch` error-path tests with `.rejects.toMatchObject()` — `createDirectory`, `writeFile`, `readFile`, and `moveFile` error tests.
