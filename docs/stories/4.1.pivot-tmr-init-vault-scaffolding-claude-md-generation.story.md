### `promptMinimalOnboarding` — Implementation Pattern

```typescript
export interface MinimalOnboardingAnswers {
  name: string;
  email: string;
  role: string;
  company: string;
}

export async function promptMinimalOnboarding(): Promise<MinimalOnboardingAnswers> {
  return inquirer.prompt<MinimalOnboardingAnswers>([
    {
      type: 'input',
      name: 'name',
      message: 'Your full name:',
      validate: (v: string): ValidateResult => v.trim().length > 0 ? true : 'Name cannot be empty',
    },
    {
      type: 'input',
      name: 'email',
      message: 'Your work email:',
      validate: (v: string): ValidateResult => v.includes('@') ? true : 'Must be a valid email address',
    },
    {
      type: 'input',
      name: 'role',
      message: 'Your current role / title:',
      validate: (v: string): ValidateResult => v.trim().length > 0 ? true : 'Role cannot be empty',
    },
    {
      type: 'input',
      name: 'company',
      message: 'Your company / domain (e.g. acme.com):',
      validate: (v: string): ValidateResult => v.trim().length > 0 ? true : 'Company cannot be empty',
    },
  ]);
}
```

### `InitCommand.run()` — New Skeleton

```typescript
async run(): Promise<void> {
  this.displayWelcomeBanner();

  const workspacePath = await promptWorkspacePath();
  const answers = await promptMinimalOnboarding();

  const scaffoldSpinner = ora('Creating workspace…').start();
  await buildWorkspaceStructure(workspacePath);

  // Write tasks.md (new)
  const tasksPath = join(workspacePath, 'my-tasks', 'tasks.md');
  const tasksExists = await fileSystemService.exists(tasksPath);
  if (!tasksExists) {
    await fileSystemService.writeFile(tasksPath, '# Tasks\n\n<!-- Backlog: add tasks here — tmr process will append extracted tasks automatically -->\n');
  }

  // Write period task files (idempotent)
  const taskPeriods: TaskPeriod[] = ['today', 'this-week', 'this-month', 'this-quarter'];
  await Promise.all(
    taskPeriods.map(async (period) => {
      const filePath = join(workspacePath, 'my-tasks', `${period}.md`);
      const exists = await fileSystemService.exists(filePath);
      if (!exists) {
        await fileSystemService.writeFile(filePath, generateTaskFileTemplate(period));
      }
    }),
  );

  scaffoldSpinner.succeed('Workspace ready');

  const claudeSpinner = ora('Generating CLAUDE.md…').start();
  await fileSystemService.writeFile(
    join(workspacePath, 'CLAUDE.md'),
    generateClaudeMd(answers),
  );
  claudeSpinner.succeed('CLAUDE.md generated');

  const pluginSpinner = ora('Downloading Obsidian plugins…').start();
  await obsidianPluginService.installPlugins(workspacePath);
  pluginSpinner.succeed('Obsidian plugins installed');

  this.displayNextSteps(workspacePath);
}
```

Implementation is clean and well-structured. `InitCommand` has been successfully reduced from 437 to ~87 LOC — the pivot is clear and purposeful. `generateClaudeMd` is a pure function with no side effects, making it highly testable. `ObsidianPluginService` change is minimal and correct. ### Refactoring Performed

- **File**: `src/commands/init.command.ts`
  - **Change**: Unified `tasks.md` and the 4 period file writes into a single `Promise.all` over `allTaskFiles` (an array of `[filename, content]` pairs).
  - **Why**: The original code wrote `tasks.md` serially (2 awaits: `exists` + conditional `writeFile`) then ran the 4 period files concurrently. This meant `tasks.md` was always sequentially blocking the period writes. All 5 files are independent — there is no ordering requirement.
  - **How**: Built an `allTaskFiles: Array<[string, string]>` array containing `['tasks.md', TASKS_MD_TEMPLATE]` plus the 4 period entries, then mapped into a single `Promise.all`. All 5 existence checks and conditional writes now run concurrently.

- **File**: `tests/services/claude-md.generator.test.ts` (new)
  - **Change**: Added 17 unit tests for `generateClaudeMd()` covering identity fields, section structure, vault table entries, output format, and determinism.
  - **Why**: `generateClaudeMd` had no dedicated unit test file. Its behavior was only verified indirectly through `init.command.test.ts` content assertions. Pure functions benefit most from direct unit tests — they are fast, deterministic, and pinpoint regressions.
  - **How**: Straightforward `describe`/`it` tests using Jest; no mocking required since the function is pure.
