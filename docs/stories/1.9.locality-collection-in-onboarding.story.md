# Story 1.9: Locality Collection in Onboarding

## Status

Done

## Story

**As a** new user running `tmr init`,
**I want** to inform my location (city, state and country) during the setup wizard,
**so that** my workspace profile reflects where I work and can be used for context in future features.

## Acceptance Criteria

1. The manager profile prompt step includes a new field: **Location** collected as a single free-text line.
   - Placeholder hint: `e.g. São Paulo, SP, Brasil`
   - The field is **optional** — pressing Enter with no value skips it gracefully (empty string stored).
2. The `ManagerProfile` TypeScript interface in `src/types/onboarding.types.ts` gains a `location?: string` field.
3. The `OnboardingData` type reflects the updated `ManagerProfile` (no additional change needed beyond AC 2).
4. The `generateCareerProfile()` template function in `src/templates/onboarding.templates.ts` includes `location` in the YAML frontmatter of `my-career/profile.md`:
   ```yaml
   location: São Paulo, SP, Brasil
   ```
   If the field is empty/undefined, the frontmatter key is omitted entirely (not written as `location: `).
5. The prompt appears after the `role` field and before the prompt block ends — no other prompt order changes.
6. Unit tests in `tests/commands/init.command.test.ts` are updated to include `location` in the mock answers sequence.
7. The integration test in `tests/integration/init.integration.test.ts` is updated accordingly.
8. `location` is **not** a required field — the test suite covers both the filled and empty cases.
