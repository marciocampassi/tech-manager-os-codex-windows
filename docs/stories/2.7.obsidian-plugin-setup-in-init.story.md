# Story 2.7: Obsidian Plugin Setup in `tmr init`

## Status

Ready for Development

## Story

**As a** manager setting up my workspace,
**I want** `tmr init` to automatically install Obsidian plugins into the workspace vault,
**so that** when I open the folder as an Obsidian vault, the plugins are already configured and ready to use without manual installation.

## Acceptance Criteria

1. **`.obsidian/plugins/` directory structure created:**
   - `{workspacePath}/.obsidian/plugins/obsidian-git/`
   - `{workspacePath}/.obsidian/plugins/obsidian-granola-sync/`
   - `{workspacePath}/.obsidian/plugins/obsidian-terminal/`

2. **Plugin files downloaded from GitHub latest releases:**
   - For each plugin, download `main.js`, `manifest.json`, and `styles.css` (when available) from the plugin's GitHub releases page using the URL pattern:
     `https://github.com/{owner}/{repo}/releases/latest/download/{file}`
   - Plugin GitHub repositories:
     - `obsidian-git`: `https://github.com/Vinzent03/obsidian-git`
     - `obsidian-granola-sync`: `https://github.com/tomelliot/obsidian-granola-sync`
     - `obsidian-terminal`: `https://github.com/polyipseity/obsidian-terminal`
   - Files to download per plugin: `main.js`, `manifest.json`, `styles.css`
   - If a file returns 404 (e.g., `styles.css` may not exist for all plugins), skip it silently — do not fail the whole init

3. **`.obsidian/community-plugins.json` created:**
   - Content enables the three plugins:
     ```json
     ["obsidian-git", "obsidian-granola-sync", "obsidian-terminal"]
     ```

4. **`.obsidian/app.json` created (if not exists):**
   - Minimal valid Obsidian app config:
     ```json
     {}
     ```

5. **Download behavior:**
   - Downloads run concurrently (all plugins in parallel, all files within a plugin in parallel)
   - A spinner shows progress: `Downloading Obsidian plugins…`
   - On network failure for a plugin file, log a warning but continue (non-fatal)
   - On complete network failure (all downloads fail), warn the user with instructions to install manually, but do not abort init
   - Total download timeout: 30 seconds per file

6. **`ObsidianPluginService` class:**
   - Located at `src/services/obsidian-plugin.service.ts`
   - Method: `installPlugins(workspacePath: string): Promise<void>`
   - Method: `downloadPluginFile(pluginDir: string, url: string, filename: string): Promise<boolean>` — returns `true` on success, `false` on failure (404, timeout, network error)
   - Uses `node:https` or `node-fetch` / native `fetch` for HTTP requests
   - Writes files via `fileSystemService.writeFile()`
   - Unit tests cover: successful download, 404 skip, network failure graceful handling, concurrent execution

7. **Integration with `InitCommand`:**
   - `buildWorkspaceStructure()` creates the `.obsidian/plugins/{plugin-id}/` directories
   - After workspace structure is built, `ObsidianPluginService.installPlugins()` is called
   - Spinner message updated to reflect plugin download phase
   - Unit tests updated to mock `ObsidianPluginService`

8. **Next steps message updated:**
   - Add line: `Obsidian plugins installed (obsidian-git, granola-sync, terminal)`
   - Instruction: `Open ${workspacePath} as your Obsidian vault — plugins are ready`

### Fetch with Timeout Pattern

```typescript
async function downloadPluginFile(destPath: string, url: string): Promise<boolean> {
  const controller = new AbortController();
  const timeout = setTimeout(() => controller.abort(), 30_000);
  try {
    const res = await fetch(url, { signal: controller.signal });
    if (!res.ok) return false; // 404, etc.
    const buffer = Buffer.from(await res.arrayBuffer());
    await fileSystemService.writeFile(destPath, buffer.toString('binary'));
    return true;
  } catch {
    return false;
  } finally {
    clearTimeout(timeout);
  }
}
```
