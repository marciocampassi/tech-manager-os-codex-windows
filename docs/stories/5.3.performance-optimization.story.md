### AC5 ‚Äî Retry Logic: Already Implemented, Needs Tuning

`BaseAIProvider.withRetry()` in `src/providers/base-ai-provider.ts` already implements exponential backoff:

```typescript
protected async withRetry<T>(fn: () => Promise<T>, maxRetries = 3): Promise<T> {
  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    try {
      return await fn();
    } catch (err) {
      if (!this.isRetryable(err) || attempt === maxRetries) throw err;
      const delay = Math.min(1000 * Math.pow(2, attempt), 10_000);
      await this.sleep(delay);
    }
  }
  throw new Error('withRetry: exhausted all attempts');
}

private isRetryable(err: unknown): boolean {
  const status = (err as { status?: number }).status;
  return status === 429 || (typeof status === 'number' && status >= 500);
}
```

**Change needed:** Architecture spec (`docs/architecture/error-handling-strategy.md`) requires 5 retries and 16s max:
- Change `maxRetries = 3` ‚Üí `maxRetries = 5`
- Change `10_000` ‚Üí `16_000`

The `isRetryable` logic is already correct: 429 and any `>= 500` (which includes 529 ‚Äî Claude overloaded). No change needed.

### AC4 ‚Äî Atomic Writes: Already Fully Implemented

`FileSystemService.writeFile()` already uses atomic temp-then-rename:

```typescript
async writeFile(filePath: string, content: string): Promise<void> {
  const tmpPath = `${filePath}.${process.pid}.tmp`;
  try {
    await fs.ensureDir(path.dirname(filePath));
    await fs.writeFile(tmpPath, content, 'utf8');
    await fs.rename(tmpPath, filePath);  // atomic on POSIX same-FS
  } catch (err) {
    await fs.remove(tmpPath).catch(() => {});
    throw new FileSystemError(/* ... */);
  }
}
```

No code changes needed. Task 3 is documentation only.

### AC3 ‚Äî `ora` Spinners: What to Add

`ora` (9.3.0) is already installed. The slow operations in `InboxProcessService.run()` are:
- **Step 3 (Categorization)**: calls AI once per file ‚Äî this is the primary bottleneck
- **Step 4 (Task extraction)**: calls AI once for all files together

Steps 1, 2, and 5 are fast (file I/O only, <<500ms even for 20 files).

**No-op shim pattern for `--plain` mode:**

```typescript
function makeSpinner(text: string, plain: boolean) {
  if (plain) {
    process.stdout.write(`${text}\n`);
    return {
      set text(_: string) {},
      succeed(msg: string): void { process.stdout.write(`${msg}\n`); },
      fail(msg: string): void { process.stdout.write(`${msg}\n`); },
      stop(): void {},
    };
  }
  return ora({ text, isSilent: false }).start();
}
```

Usage in step 3:

```typescript
const spinner = makeSpinner('[3/5] Categorizing files‚Ä¶', options.plain);

  // ...

spinner.succeed(`[3/5] Categorized ${summary.filesCategorizedOk} / ${files.length} files`);
```
