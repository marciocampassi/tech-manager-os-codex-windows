## Acceptance Criteria

1. `package.json` finalized with correct name, version, and binary (`tmr`)

3. Binary executable works: `npx tech-manager-os init`
4. `.npmignore` configured to exclude source, tests, and dev files
5. npm package tested locally with `npm pack` and `npm install -g`
6. Skills registry endpoint configured and `tmr install tmr-inbox` works against published package
7. Ready for `npm publish`

### Current tsup Config State

[Source: `tsup.config.ts`]

```ts
export default defineConfig({
  entry: ['src/cli.ts'],
  format: ['esm'],
  splitting: true,
  dts: true,
  clean: true,
  sourcemap: true,
  target: 'node18',
  outDir: 'dist',
```

The shebang banner is applied to all outputs via `banner.js`. With `splitting: true`, tsup produces multiple chunk files — the shebang is added to the entry output (`dist/cli.js`). This is correct for the `bin` entry.

**Note on `dts: true`**: TypeScript declarations are generated into `dist/`. These are harmless to publish but can be excluded from `.npmignore` if size matters. Leave them in by default.

### Testing

[Source: `docs/architecture/test-strategy-and-standards.md`]

- Framework: **Jest** (confirmed active in project — not Vitest; architecture doc is aspirational)
- New test files: `tests/packaging/package-json.test.ts`, `tests/packaging/build-output.test.ts`, `tests/packaging/npmignore.test.ts`
- Pattern: AAA (Arrange, Act, Assert)
- No mocking needed for packaging tests — these are structural/file-system assertions
- Build output tests: run after `npm run build` is confirmed via `beforeAll` checks using `fs.existsSync`
- Coverage requirement: 80% overall minimum; packaging tests are additive

### Testing Standards

- Test location: `tests/packaging/` (new subdirectory, mirrors packaging domain)
- Test framework: Jest (ESM mode via `NODE_OPTIONS=--experimental-vm-modules`)
- Assertions style: `expect(value).toBe(expected)` / `expect(value).toContain(expected)`
- For file-system tests: use `node:fs` and `node:path` directly — no mocking required
- For npm pack dry-run: spawn child process using `node:child_process` `execSync` and capture stdout
