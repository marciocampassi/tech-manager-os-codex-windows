### Language & Runtime Requirements
- **TypeScript 5.x** — strict mode is mandatory
- **Node.js 18+** (LTS minimum)
- **ES Modules (ESM)** — set `"type": "module"` in `package.json` and `"module": "ESNext"` in `tsconfig.json`
- ESM requires specific Jest config — use `ts-jest` with `extensionsToTreatAsEsm: ['.ts']`

[Source: docs/prd/technical-assumptions.md#language--runtime]

### Build Tool: `tsup` (not `tsc` directly)
- Use `tsup` for production builds — it is faster and handles ESM edge cases better than raw `tsc`
- `tsc` is still used for type checking (`tsc --noEmit`)
- Build output goes to `dist/`, entry is `src/cli.ts`

[Source: docs/prd/technical-assumptions.md#build--distribution]

### Refactoring Performed

- **File:** `eslint.config.js` (created) + `.eslintrc.json` / `.eslintignore` (deleted)
  - **Change:** Migrated ESLint from legacy `.eslintrc.json` format to ESLint 10 flat config (`eslint.config.js`). Installed `@eslint/js`. Removed `.eslintignore` (ignores now inline in flat config).
  - **Why:** ESLint 10 drops support for `.eslintrc.*` files entirely — `npm run lint` was failing with exit code 2.
  - **How:** Flat config uses `export default [...]` array of config objects, giving finer-grained control per file glob. Ignores are declared inside the config array instead of a separate file.

- **File:** `src/cli.ts`
  - **Change:** Removed `#!/usr/bin/env node` shebang from source file.
  - **Why:** `tsup.config.ts` already declares `banner: { js: '#!/usr/bin/env node' }`, causing `dist/cli.js` to open with two consecutive shebangs. Only the first shebang is honoured by the OS; the second appears as invalid syntax.
  - **How:** Shebang belongs in the compiled binary, not the TypeScript source. `tsup` handles it via the `banner` config.

- **File:** `package.json`
  - **Change:** Updated `lint` script from `eslint src --ext .ts` to `eslint src`.
  - **Why:** `--ext` flag is removed in ESLint 10. File filtering is now handled by the `files` glob in `eslint.config.js`.
  - **How:** ESLint flat config's `files: ['src/**/*.ts']` replaces the `--ext` flag behaviour.
