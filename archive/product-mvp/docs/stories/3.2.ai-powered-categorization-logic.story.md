### AI Provider Integration

**CRITICAL:** AI calls MUST go through `AIProvider` interface injected via constructor — NEVER use `AIProviderFactory` or direct SDK calls inside the service (keep it testable). The caller (future `tmr process` command in Story 3.6) is responsible for constructing the provider.

```typescript
import type { AIProvider } from '../providers/ai-provider.interface.js';
```

Available interface method:

```typescript
generateText(prompt: string, options?: GenerateOptions): Promise<string>
// options: { model?, maxTokens?, temperature?, systemPrompt? }
```

Use `maxTokens: 1024` (sufficient for categorization JSON) and pass a `systemPrompt` constant.

Do NOT call `streamText` — categorization requires the complete JSON before processing.

[Source: src/providers/ai-provider.interface.ts]

### Confidence Threshold

The default threshold is `0.75`, sourced from `configService.getConfidenceThreshold()` (which itself defaults to `CONFIG_DEFAULTS.confidence_threshold = 0.75`). For this service, accept the threshold as a constructor parameter (default `0.75`) — do NOT import `configService` directly. The caller injects the threshold, keeping the service pure and testable.

```typescript
export const LOW_CONFIDENCE_THRESHOLD = 0.75;

export class CategorizationService {
  constructor(
    private readonly _ai: AIProvider,
    private readonly _confidenceThreshold: number = LOW_CONFIDENCE_THRESHOLD,
  ) {}
}
```

### AI Prompt Design

The service builds a two-part prompt:

**System prompt (constant `SYSTEM_PROMPT`):**

```
You are an expert categorization assistant for an engineering manager's workspace.
Analyze the provided note and respond with ONLY a valid JSON object — no markdown, no explanation, no code fences.
The JSON must have this exact structure:
{
  "type": "<one of: 1on1_session|feedback_positive|feedback_constructive|pip_concern|project_status|project_risk|team_meeting|leadership_meeting|candidate_interview|general_note>",
  "members": ["array of member names explicitly mentioned"],
  "projects": ["array of project names explicitly mentioned"],
  "insights": { "<member_or_project_name>": ["key insight 1", "key insight 2"] },
  "destinations": ["workspace-relative path for filing this note, e.g. my-teams/alpha/john@co.com/1on1s/"],
  "suggestedActions": ["specific follow-up action item"],
  "confidence": 0.95
}
Rules:
- "type" must be one of the exact values listed above
- "confidence" must be a number 0-1 representing your certainty
- "destinations" must use known member emails and team names from the provided context
- If uncertain about routing, lower confidence score; do not guess member emails
```

**User prompt (built per call):**

```
Known team members:
- John Doe <john.doe@company.com> (team: alpha)
...

Known projects:
- api-redesign (API Redesign Initiative)
...

Note filepath: /workspace/inbox/meeting-with-john.md
Note content:
<file content here>
```

### JSON Parsing and Validation

The AI response is an external input and MUST be validated manually (no Zod in project):

```typescript
private _parseAIResponse(raw: string): Result<CategorizationResult> {
  let parsed: unknown;
  try {
    parsed = JSON.parse(raw.trim());
  } catch {
    return { success: false, error: `AI returned invalid JSON: ${raw.slice(0, 100)}` };
  }
  // Type-guard checks for required fields
  if (!isCategorizationPayload(parsed)) {
    return { success: false, error: 'AI response missing required fields or invalid types' };
  }
  // needsReview is set by the service, not the AI
  return { success: true, data: { ...parsed, needsReview: parsed.confidence < this._confidenceThreshold } };
}
```

Implement `isCategorizationPayload(obj: unknown): obj is RawAIPayload` as a private type-guard that checks:
- `typeof obj === 'object' && obj !== null`
- `type` is a string AND is one of the 10 valid `NoteType` values
- `members` is `string[]`
- `projects` is `string[]`
- `insights` is `Record<string, string[]>`
- `destinations` is `string[]`
- `suggestedActions` is `string[]`
- `confidence` is `number` in range [0, 1]

Do NOT use Zod — it is not installed. Use explicit `Array.isArray` and `typeof` checks.

[Source: docs/architecture/coding-standards.md#critical-rules ("All external inputs must be validated")]

### Error Handling

- AI `generateText` throws → catch and return `{ success: false, error: err.message }`
- AI returns invalid JSON → return `{ success: false, error: 'AI returned invalid JSON: ...' }`
- AI returns valid JSON but wrong shape → return `{ success: false, error: 'AI response missing required fields...' }`
- Never throw from `categorize()` — always return `Result<CategorizationResult>`
