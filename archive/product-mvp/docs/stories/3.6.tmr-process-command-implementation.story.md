# Story 3.6: `tmr process` Command Implementation

## Status

Ready for Review

## Story

**As a** leader,
**I want** one command to process all inbox files,
**so that** my workspace stays current with minimal effort.

## Acceptance Criteria

1. `tmr process` orchestrates: Scan → Categorize → Update Contexts → Extract Tasks → Organize Files
2. Progress indicators for each phase
3. Summary output:
   - Files processed
   - Contexts updated (people, projects)
   - Tasks extracted
   - Suggested actions
4. Handles errors gracefully (partial progress saved)
5. Optional flags: `--dry-run`, `--verbose`
6. Integration test validates full process workflow

## Tasks / Subtasks

- [x] Task 2: `InboxProcessService` orchestrator (AC: 1, 3, 4)
  - [x] Add `src/services/inbox-process.service.ts` — constructor DI: `InboxService`, `CategorizationService`, `ContextService`, `TaskService`, `FileOrganizationService`, `TeamService`, `ProjectService`, `FileSystemService`
  - [x] `run(workspaceRoot: string, options: ProcessRunOptions): Promise<Result<ProcessSummary>>` — never throw; return `Result`
  - [x] **Phase Scan:** `inboxService.scanInbox` — if empty, return success with zero counts
  - [x] **Load context:** `loadCategorizationContext` — on failure, return `{ success: false, error }`
  - [x] **Phase Categorize:** for each inbox file (oldest first), `categorize(file, ctx)`; record failures per file; count `needsReview` for summary
  - [x] **Phase Update contexts:** for each successful categorization, map `CategorizationResult.insights` keys to member (match `name` or `email` case-insensitively against loader members) or project (match slug / `displayName`); call `ContextService.updateContext` / `updateProjectContext` with string arrays; skip unknown keys; if `dryRun`, skip writes but count would-be updates
  - [x] **Phase Tasks:** `taskService.extractTasks(files, workspaceRoot)` on full scanned file list; if `dryRun`, skip; record counts / error
  - [x] **Phase Organize:** for each file with successful categorization, `fileOrganizationService.organizeFile(file, workspaceRoot, destinations)`; if `dryRun`, skip; record per-file errors without aborting the rest
