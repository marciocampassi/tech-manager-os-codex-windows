# Story 4.5: Skill Install/Update Mechanism

## Status

Ready for Done

## Story

**As a** leader,
**I want** to install and update skills from the CLI,
**so that** I receive new capabilities without manually copying files.

## Acceptance Criteria

1. `tmr install <skill-name>` command:
   - Downloads the named skill from the official registry (GitHub releases or npm package assets)
   - Installs it into `.claude/skills/<skill-name>/SKILL.md` in the current vault directory
   - Displays a confirmation with the installed skill name and version
   - Error handling covers: skill not found, network failure, already installed (offers to update instead)
2. `tmr update` command:
   - Scans `.claude/skills/` to detect all installed skills
   - Checks for newer versions in the registry
   - Updates all skills that have newer versions available
   - Reports per skill: updated to vX.Y or already up to date
3. `tmr install tmr-inbox` works as the canonical first-use case
4. Skills are installed without any user-specific values ÔÇö the skill reads from `CLAUDE.md` at runtime
5. Unit tests cover: install happy path, already-installed detection, update detection, no-skills-installed case

### Registry Design

The official skill registry is this GitHub repository. Skills are fetched as raw GitHub content from the `main` branch:

```
REGISTRY_BASE_URL = 'https://raw.githubusercontent.com/marlonvidal/tech-manager-os/main/skills'
Skill URL pattern: {REGISTRY_BASE_URL}/{skillName}/SKILL.md
tmr-inbox URL:     https://raw.githubusercontent.com/marlonvidal/tech-manager-os/main/skills/tmr-inbox/SKILL.md
```

**Version detection:** The SKILL.md file carries a version comment on line 1:
```
<!-- version: 1.0.0 -->
```
Parse this via regex: `/<!--\s*version:\s*(\S+)\s*-->/`. If absent, treat version as `'0.0.0'`.

**Manifest file:** `.claude/skills/skill-manifest.json` ÔÇö JSON array of `SkillManifestEntry`:
```json
[
  { "name": "tmr-inbox", "version": "1.0.0", "installedAt": "2026-04-21T10:00:00.000Z" }
]
```

**Network implementation:** Use Node.js built-in `https.get()` (no extra dependency). Wrap in a `Promise<string>` helper. On non-200 status code, reject with an appropriate error. Set a 10-second timeout.

**Version comparison:** Compare as arrays of integers split by `.`. No semver library needed.

### SkillRegistryService API

```typescript
interface SkillManifestEntry {
  name: string;
  version: string;
  installedAt: string; // ISO 8601
}

class SkillRegistryService {
  constructor(private readonly workspaceRoot: string) {}

  getRegistryUrl(skillName: string): string
  fetchSkillContent(skillName: string): Promise<Result<{ content: string; version: string }>>
  readManifest(): SkillManifestEntry[]
  writeManifest(entries: SkillManifestEntry[]): void

  getInstalledVersion(skillName: string): string | undefined
  installSkill(skillName: string, content: string, version: string): void

}
```

**`installSkill` path:** `path.join(workspaceRoot, '.claude', 'skills', skillName, 'SKILL.md')`
**Manifest path:** `path.join(workspaceRoot, '.claude', 'skills', 'skill-manifest.json')`

Ensure parent directory exists before writing SKILL.md: `fs.mkdirSync(dir, { recursive: true })`.

### Version Comparison Helper

```typescript
function isNewerVersion(candidate: string, installed: string): boolean {
  const parse = (v: string): number[] => v.split('.').map(Number);
  const [ca, cb, cc] = parse(candidate);
  const [ia, ib, ic] = parse(installed);
  if (ca !== ia) return ca > ia;
  if (cb !== ib) return cb > ib;
  return cc > ic;
}
```

### Code Quality Assessment

High-quality, well-structured implementation. Story 4.5 adds two new CLI commands and a service layer with no modifications to existing code (except registering the commands in `cli.ts`). The design is clean and follows established project patterns throughout.

`SkillRegistryService` is single-responsibility and well-encapsulated. The `Result<T>` discriminated union return type on `fetchSkillContent` provides type-safe error handling without throwing. The manifest upsert pattern (filter-then-push) is correct and predictable. The `fetchUrl` helper is appropriately scoped as a module-level private function. The 10-second request timeout prevents hanging on slow connections.

`install.command.ts` and `update.command.ts` both follow the `process.command.ts` pattern exactly ÔÇö `configService.initialize()` at entry, `getWorkspaceRoot()` for path resolution, chalk helpers with `opts.plain` respect, `process.stdout.write` (no `console.log`).

One refactoring was applied: the original `runInstall()` called `isInstalled()` then `getInstalledVersion()` as two separate manifest reads. Refactored to a single `getInstalledVersion()` call with `!== undefined` guard, eliminating a redundant filesystem read. The install tests were updated to explicitly reset `mockGetInstalledVersion` where `clearAllMocks()` didn't restore the initial implementation after `mockReturnValue` calls in prior tests.

### Refactoring Performed

- **File**: `src/commands/install.command.ts`
  - **Change**: Replaced `isInstalled()` + `getInstalledVersion()` (two manifest reads) with a single `getInstalledVersion()` call; guards on `!== undefined` for the already-installed check
  - **Why**: `isInstalled()` reads the manifest and returns a boolean; `getInstalledVersion()` reads it again and returns the version. Combining into one call halves manifest I/O for the already-installed path
  - **How**: `const installedVersion = registry.getInstalledVersion(skillName); if (installedVersion !== undefined && !opts.force) { ... }`

- **File**: `tests/commands/install.command.test.ts`
  - **Change**: Added `mockGetInstalledVersion.mockReturnValue(undefined)` in Tests A, D, E ÔÇö tests that expect "not installed" behaviour
  - **Why**: `jest.clearAllMocks()` (used in `beforeEach`) resets call counts but not `mockReturnValue` overrides; Test B sets `mockGetInstalledVersion.mockReturnValue('1.0.0')` which persisted into later tests after the refactor switched from `isInstalled` to `getInstalledVersion`
  - **How**: Explicit `mockGetInstalledVersion.mockReturnValue(undefined)` in each affected test
