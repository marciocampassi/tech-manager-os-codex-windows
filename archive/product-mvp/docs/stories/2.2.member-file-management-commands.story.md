## Story

**As a** manager,
**I want** CLI commands to create member-related files and auto-update profile sections,
**so that** Obsidian wiki-links stay current without manual editing.

## Acceptance Criteria

1. **`tmr member add 1on1 <email> [--date=<date>]`** creates `1on1s/{date}-{email}-1on1.md` inside the member directory and appends a wiki-link to the `## 1on1s` section of `{email}.md`.
2. **`tmr member add feedback <email> [--date=<date>]`** creates `feedback/{date}-{email}-feedback.md` and appends a wiki-link to `## Feedbacks`.
3. **`tmr member add assessment <email> [--date=<date>]`** creates `assessments/{date}-{email}-assessment.md` and appends to `## Assessments`.
4. **`tmr member add performance-review <email> [--date=<date>]`** creates `performance-reviews/{date}-{email}-review.md` and appends to `## Performance Reviews`.
5. All commands: if `<email>` is omitted, prompt interactively; if member not found, print a clear error with suggestion to run `tmr team add` first.
6. All commands: `--date` defaults to today (`YYYY-MM-DD`); `--no-edit` flag suppresses auto-opening the created file.
7. **`SectionParserService`** correctly finds a markdown `##` section, appends content before the next `##` section (or EOF), and creates the section if it doesn't exist.
8. **`TemplateService`** returns the correct markdown template for each file type with date and email placeholders replaced.
9. All commands have **unit tests** and an **integration test** validates the full create-and-link workflow on a temp workspace.

### CLI Command Structure

```
tmr member add <type> [email] [--date YYYY-MM-DD] [--no-edit]
```

Commander.js structure â€” `member` is the parent command, `add` is a subcommand, `<type>` and `[email]` are positional arguments on `add`:

```typescript
export function createMemberCommand(): Command {
  const cmd = new Command('member').description('manage member-related files');
  cmd.command('add <type> [email]')
    .description('add a file for a team member (type: 1on1|feedback|assessment|performance-review)')
    .option('--date <date>', 'date for the file (YYYY-MM-DD)', todayIso())
    .option('--no-edit', 'do not open the created file')
    .action(async (type, email, opts) => { ... });
  return cmd;
}
```

> **Note:** PRD shows `tmr member <email> add 1on1` but Commander.js doesn't support positional args + subcommands on the same command level cleanly. The implementation uses `tmr member add <type> [email]` which is functionally equivalent and more idiomatic.

### `SectionParserService` â€” Key Algorithm

Operates on the **content string** (pure function â€” no IO), then the caller writes atomically:

```typescript
appendToSection(content: string, sectionName: string, appendLine: string): string {
  const lines = content.split('\n');
  const secIdx = lines.findIndex(l => l === `## ${sectionName}`);

  if (secIdx === -1) {
    // Section missing â€” append it at end
    return content.trimEnd() + `\n\n## ${sectionName}\n\n${appendLine}\n`;
  }

  // Find where the next ## section starts (or end of file)
  let nextSecIdx = lines.findIndex((l, i) => i > secIdx && l.startsWith('## '));
  if (nextSecIdx === -1) nextSecIdx = lines.length;

  // Build updated content
  const before = lines.slice(0, nextSecIdx).join('\n').trimEnd();
  const after = lines.slice(nextSecIdx).join('\n');
  return `${before}\n${appendLine}\n${after.length > 0 ? '\n' + after : ''}`;
}
```

The `appendToFile` method reads â†’ transforms â†’ writes atomically via `FileSystemService`.

### Opening in Editor

Default: attempt to open the created file using `$EDITOR` env var; fall back to `open` (macOS) / `xdg-open` (Linux/WSL); if unavailable, print the file path. With `--no-edit`: skip and print path only.

```typescript
import { exec } from 'node:child_process';

function openInEditor(filePath: string): void {
  const editor = process.env['EDITOR'];
  if (editor) {
    exec(`${editor} "${filePath}"`);
  } else if (process.platform === 'darwin') {
    exec(`open "${filePath}"`);
  } else {
    exec(`xdg-open "${filePath}"`);
  }
}
```

Wrap in try/catch â€” editor failure should not fail the command. Always print the path regardless.
